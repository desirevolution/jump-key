# JumpKey UI Design Recipe
## A Design System Guide for Building New Components

This document describes the visual and UX principles used throughout the JumpKey dashboard UI.  
The goal is to keep future components consistent with the existing dark, self-hosted dashboard style.

---

# 1. Core Design Philosophy

JumpKey follows a:

- **Dark-first**
- **Minimal**
- **Developer-oriented**
- **Keyboard-friendly**
- **Self-hosted dashboard aesthetic**

The interface should feel like:

- a modern terminal
- a professional admin dashboard
- a personal command center

Avoid:
- excessive decoration
- bright gradients
- unnecessary animations
- large rounded "marketing style" cards
- heavy shadows

Prefer:
- information density
- clear hierarchy
- subtle interaction feedback
- predictable behavior

---

Every new JumpKey component should look like it has always been part of the system.
